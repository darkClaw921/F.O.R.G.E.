# push.rs

tmux-web/src/push.rs — Web Push доставка (Фаза 3): VAPID-JWT (ES256) + RFC 8291/8188 aes128gcm шифрование + воркер attention_watcher + транспорт send_one/send_to_all. Тесты в #[cfg(test)] mod tests расширены краевыми случаями: ECE roundtrip (пустой/UTF-8/бинарь/граница RECORD_SIZE/oversize-без-паники), негативы ключей подписки (битый base64url, padding, неверная длина 64/66, prefix!=0x04, точка не на кривой, auth!=16), свежесть salt/server-key (10 вызовов уникальны), целостность GCM (подмена ciphertext/salt/AAD/порядка ua||as ломает decrypt), изоляция per-subscription. VAPID-JWT: no-pad сегменты, P1363 raw 64-байт подпись+verify своим pubkey, exp в [now, now+24h], aud==origin endpoint, непустой sub, header k==public_key. Транспорт (wiremock, #[tokio::test multi_thread]): 404/410→prune, 401/403→Auth-без-prune, 500/connection-refused→Transient-без-prune, 2xx→sent++, смесь живых/мёртвых, битый ключ→Crypto-до-HTTP, обязательные заголовки (Content-Encoding/TTL/Urgency/Authorization vapid t=), opt-in lazy file creation, конкурентность send_to_all с upsert/remove. Edge-trigger воркера через зеркальную newly_attention: стабильный true не повторяет пуш, пустые входы без триггера, смешанные фазы сессий.
