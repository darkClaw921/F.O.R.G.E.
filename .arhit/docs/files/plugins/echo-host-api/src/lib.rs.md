# plugins/echo-host-api/src/lib.rs

Plugin boundary крейт для Echo. Определяет trait HostApi через async_trait для dyn-совместимости. После Phase 4 (remove-projects-concept) методы list_projects() / active_project_id() удалены вместе с ProjectInfo DTO — host больше не транслирует понятие «проект» в Echo плагин. Остались методы: list_sessions() → Vec<SessionInfo>, capture_pane_full(session, lines) → String, auth_token() → Option<String>. SessionInfo (упрощённый view tmux-сессии): {name, windows, panes}. Echo продолжает хранить опциональный project_id в своих SQLite-таблицах (chat_sessions.project_id, memories.project_id) как непрозрачный soft-FK label, но валидация и enumeration снаружи плагина не предоставляются. Конкретная реализация trait'а — tmux-web/src/echo_host.rs::EchoHostAdapter.
