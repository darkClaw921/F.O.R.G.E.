# main.rs

Точка входа devforge (axum + tmux + WS). PWA wire-up (opt-in --pwa): pwa_ctx: Option<PwaCtx{vapid,subs}> инициализируется при pwa_enabled (~457) — VapidStore::load_or_generate + PushSubscriptionStore::new в ~/.forge; перемещается в AppState.pwa (~586). Роуты /api/pwa/config + /api/push/{subscribe,unsubscribe,test} регистрируются только при app_state.pwa.is_some() до auth-layer (~800). Push-воркер доставки: сразу после tokio::spawn(attention::watcher_loop(...)) условный спавн — if let Some(ctx)=app_state.pwa.as_ref() { tokio::spawn(push::attention_watcher(attention.clone(), ctx.subs.clone(), ctx.vapid.clone(), '/'.to_string())) }. Без --pwa pwa==None -> воркер не спавнится, push-роуты 404 (opt-in инвариант). mod push/push_store без allow(dead_code) (Фаза 3 потребляет все элементы).
