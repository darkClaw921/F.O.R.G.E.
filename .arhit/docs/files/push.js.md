# push.js

PWA Web Push UI (Фаза 5, tmux-web/static/js/pwa/push.js). Лениво из bootstrap при enabled. initPush() (идемпотентна, guard pushSupported(): serviceWorker+PushManager+Notification): MutationObserver на body ловит открытие Settings-модала (#ps-panel-notifications вкладка Notifications), injectPushSection вставляет секцию .pwa-push-section с тогглом #pwa-push-toggle + статусом + кнопкой #pwa-push-test-btn. ВКЛЮЧЕНИЕ: Notification.requestPermission() вызывается СИНХРОННО в стеке обработчика change (iOS user-gesture, без предшествующего await) -> .then: при granted reg.pushManager.subscribe({userVisibleOnly:true, applicationServerKey: urlBase64ToUint8Array(vapid)}) -> POST /api/push/subscribe тело=subscription.toJSON() ({endpoint, keys:{p256dh,auth}}; backend SubscribeReq без deny_unknown_fields игнорит expirationTime), Content-Type application/json + credentials same-origin (csrf + remote-token через подменённый fetch core/auth.js). При неудаче backend — откат sub.unsubscribe(). ВЫКЛЮЧЕНИЕ: getSubscription -> sub.unsubscribe() + POST /api/push/unsubscribe({endpoint}). refreshState при открытии: iOS&!standalone -> toggle disabled + подсказка установить; Notification.permission==denied -> disabled+warn; иначе getSubscription() определяет checked/test-btn. Кнопка теста POST /api/push/test. vapidPublicKey из window.__FORGE_PWA. Экспорты: initPush, urlBase64ToUint8Array (base64url->base64 +padding -> atob -> Uint8Array). Импортирует isIos/isStandalone из install.js.
