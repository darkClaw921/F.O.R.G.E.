# routes/daily_reports.rs

REST API «Сводки дня» под /api/echo/daily-reports (Phase 2 фичи forge-b4q). Три хендлера на router(): GET /api/echo/daily-reports?limit=N — список последних N отчётов (daily_reports::list, DEFAULT_LIMIT=30, clamp 1..365), возвращает {items:[DailyReport]}; GET /api/echo/daily-reports/:day — отчёт за локальный день YYYY-MM-DD (daily_reports::get_by_day), валидирует формат (400 на битый), 404 если записи нет; POST /api/echo/daily-reports/generate body {day?:YYYY-MM-DD} — синхронная генерация через daily_report::generate_report(source=manual) с tokio::time::timeout(GENERATE_TIMEOUT=90s), default day=сегодня по chrono::Local, host берётся из state.host.get(), 200 {id,day,content}, 504 при таймауте, 400 на битый day. Ошибки — единый ApiError(StatusCode,String): IntoResponse сериализует {error:...} (та же конвенция, что routes::memory). Регистрируется через routes/mod.rs::build_router .merge(daily_reports::router()). Unit-тесты (mock-CLI + build_router + oneshot): generate 400 на битый day, 200 на explicit/default day, get_by_day 404/400, list+get roundtrip.
