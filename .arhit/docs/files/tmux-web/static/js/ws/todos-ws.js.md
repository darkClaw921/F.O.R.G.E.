# tmux-web/static/js/ws/todos-ws.js

Frontend WebSocket клиент для /ws/todos и REST helper для /api/todos. После Phase 5 фильтрация TODOs идёт по cwd-path активной tmux-сессии: query-param ?path=<abs>, а не ?project_id=. Экспорты: fetchTodos(path) — GET /api/todos?path=…; startTodosPolling/stopTodosPolling — fallback polling каждые 30s когда WS оффлайн; connectTodosWs/disconnectTodosWs — WebSocket lifecycle; syncTodosToCurrentSession — вызывается из openSession/switchSession, сравнивает state.todosCurrentPath с sess.path активной сессии и пересоединяет WS если они различаются; scheduleTodosWsReconnect — экспоненциальный backoff; handleTodosWsMessage — обработка snapshot/upsert/removed/reload событий. sessionPathOrNull() — внутренний helper, берёт sess.path из state.sessions для state.currentSession (или null если сессии нет или у неё нет cwd).
