# tmux-web/tests/frontend/sidebar_grouping.test.js

Phase 6 / forge-cca8.2 — регресс-тесты группировки sidebar. Standalone Node-test (без jest/vitest), запуск: node tmux-web/tests/frontend/sidebar_grouping.test.js. Покрывает: (1) groupSessionsByProject — сессии группируются по project_id, project_id=null/undefined → __orphan__, внутри группы сортировка по name.localeCompare, кастомный orphanKey; (2) decideVisibleOrigins — какие origin'ы рендерим при activeOrigin = 'all' | 'local' | <server_id>; (3) aggregateAllOrigins — структура {origin: {projects, sessions, online}} собирается из state.* + remote-кешей, offline-сервера присутствуют с пустыми данными; (4) applyProjectFilterInsideOrigin — двухуровневая фильтрация: __all__ внутри origin даёт группы, конкретный project_id даёт плоский список, проект отсутствующий в origin → пустой список; (5) переключение origin НЕ сбрасывает projectFilter (явное архитектурное решение); (6) legacy совместимость projectFilter='__all__'. Реплицирует pure-логику из app.js (без DOM); любое изменение контракта groupSessionsByProject/aggregateAllOrigins должно обновлять и этот файл.
