# tmux-web/src/ws_tasks.rs

WebSocket-handler /ws/tasks?path=... для realtime task-стрима. После Phase 4 (remove-projects-concept) query-параметр сменился с project_id на path. Per-connection watcher (а не shared broadcast): каждое соединение отслеживает ровно тот .beads/, который соответствует его ?path=. Wire-протокол: {kind:snapshot,data:...}, {kind:upsert,issue:...}, {kind:removed,id:...}, {kind:reload}. Heartbeat — 30s ping. resolve_active_path(state, ?path=) — fallback на state.active_path_tx.borrow() при отсутствии path. Поддержка ?server=<id> (remote-proxy через remote_proxy::proxy_websocket) при remote_mode=true. Renamed project_path → active_path в handle_socket.
