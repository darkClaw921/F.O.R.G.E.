# mobile.js

PWA мобильные улучшения (Фаза 5, tmux-web/static/js/pwa/mobile.js). Лениво из bootstrap при enabled. initMobile() (идемпотентна) добавляет body.pwa-active (гейт CSS-правил safe-area/overscroll в css/pwa.css — opt-in) и запускает 4 фичи через safe()-обёртку (try/catch-изоляция). КРИТ: (1) initKeyboardViewport — visualViewport resize/scroll -> высота клавиатуры = innerHeight - vv.height - vv.offsetTop -> CSS-var --pwa-keyboard-height + body.pwa-keyboard-open -> refitTerminal (state.fitAddon.fit() + scheduleResizeFromTerm() из terminal/xterm.js), ввод xterm над клавиатурой; (2) overscroll-behavior:contain на #terminal/#tasks-board/#tasks — в css/pwa.css под body.pwa-active (блок pull-to-refresh, не рвёт WS); (3) safe-area — #tab-bar/#settings-bar padding под нотч в css/pwa.css. NICE: initAppBadge (navigator.setAppBadge/clearAppBadge, count=число needs_attention из лёгкого поллинга /api/sessions раз в 5с, стоп при document.hidden), initWakeLock (кнопка #pwa-wakelock-toggle в #settings-bar, navigator.wakeLock.request('screen'), re-acquire на visibilitychange если wakeLockWanted), initOnlineBanner (.pwa-offline-banner + status-dot.pwa-offline по navigator.onLine + online/offline события). vibrate НЕ дублируется (уже в sw.js push-handler). Импортирует state (core/state.js), scheduleResizeFromTerm (terminal/xterm.js). Экспорт initMobile.
