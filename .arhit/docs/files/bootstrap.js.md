# bootstrap.js

Единственный ВСЕГДА-загружаемый PWA-файл devforge (tmux-web/static/js/pwa/bootstrap.js). Подключён статически из index.html. fetch('/api/pwa/config'): не-200/enabled!=true -> disablePwa() (unregister всех SW + чистка forge-* кэшей, разметку не трогает — строгий opt-out). enabled=true -> window.__FORGE_PWA={enabled,vapidPublicKey}, injectHead() (manifest/theme-color/apple-meta/apple-touch-icon/css-pwa), import register.js -> registerServiceWorker(). ФАЗА 5: loadInstall()/loadPush()/loadMobile() — ленивые import install.js/push.js/mobile.js (каждый в try/catch, вызывает init*()). handleLaunchParams(): разбирает location.search shortcuts/share-target из manifest — ?tab=tasks|echo|... -> routeTab (import tabs.js switchTab), ?view=daily-summary -> routeView (import daily-summary.js showDailySummary), ?share-target=1 +title/text/url -> routeShareTarget (switchTab echo + вставка объединённого текста в #echo-input с retry-поллингом + dispatch input event). Все маршруты ждут whenAppReady() (флаг window.__FORGE_APP_READY + событие forge:app-ready, выставляются в core/bootstrap.js по завершении init; фолбэк-поллинг 8с) — иначе switchTab/showDailySummary упадут до инициализации main.js. Без relevant-параметра дефолтное поведение не меняется. Без enabled — никаких побочных эффектов на разметку (opt-in).
