# vapid.rs

tmux-web/src/vapid.rs — VAPID P-256 keypair store (~/.forge/vapid.json, atomic tmp+rename) + ES256 подпись для Web Push. Тесты в #[cfg(test)] mod tests расширены краевыми случаями: генерация даёт приватник ровно 32 байта + публичный 65 байт 0x04-prefix; рассинхрон кэша public_key_b64 в файле → публичный ключ выводится заново из приватника (не из подменённого кэша); пустой файл→Err парсинга (не NotFound, не перегенерация); JSON без обязательных полей→Err; невалидный base64url приватника (мусор/padding/'+'/'/'); приватник длиннее 32 байт (33/64)→Err; криптографически невалидный 32-байт скаляр (0 и 0xFF*32)→Err; толерантность к whitespace вокруг приватника (.trim()); чтение каталога вместо файла (не-NotFound)→Err без генерации; идемпотентность reload не переписывает байты файла; отсутствующий родительский каталог→Err (нет auto-create, fail-soft); детерминизм derive_public_b64; public_key_b64 base64url-no-pad; Debug не печатает приватник (<redacted>); default_vapid_path с HOME→.forge/vapid.json и без HOME→Err (set/remove_var + restore в одном тесте против гонок env).
