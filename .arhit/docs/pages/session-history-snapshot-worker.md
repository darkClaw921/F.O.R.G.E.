Snapshot-воркер истории сессий (main.rs, Phase 2, epic forge-joc). Назначение: периодически и при завершении фиксировать снимок активных tmux-сессий в персистентный журнал HistoryStore, чтобы UI 'Недавние сессии' мог показать и восстановить ранее виденные сессии после kill-session/перезагрузки tmux.

КОМПОНЕНТЫ:
1. AppState.history: session_history::HistoryStore — cheap-clone обёртка (Arc<RwLock> внутри), загружается через HistoryStore::load(&themes_dir) при старте (~/.config/forge/session_history.json). Битый/отсутствующий файл → пустой стор, старт не блокируется.
2. Периодический воркер: tokio::spawn с tokio::time::interval(Duration::from_secs(3600)). interval по умолчанию срабатывает немедленно на первом тике (снимок сразу при старте), далее раз в час. На каждом тике вызывает session_history::capture_now(&history_for_worker). В spawn передаётся только клон HistoryStore (а не весь AppState) — воркеру не нужны остальные поля.
3. Graceful-shutdown хук: внутри shutdown_signal future, переданного в axum::serve(...).with_graceful_shutdown(shutdown_signal). Последовательность: wait_for_shutdown_signal() (ctrl_c или SIGTERM) → forge_echo::shutdown(&echo_for_shutdown) → tracing::info 'capturing final session history snapshot before shutdown' → session_history::capture_now(&history_for_shutdown). Гарантирует, что финальный снимок попадает в session_history.json до остановки процесса.

БИЗНЕС-ЛОГИКА: capture_now опрашивает tmux и делает upsert по ключу name+path (атомарная запись через temp+rename). Дубликатов не создаёт — повторные снимки той же сессии обновляют last_seen/windows. Восстановление сессий из журнала реализуют endpoint'ы /api/sessions/history/restore[-all] (restore_one_session). Зависимости: crate::session_history (HistoryStore, capture_now), tokio, axum graceful shutdown.
