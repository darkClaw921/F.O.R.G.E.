Модуль js/home/home.js — главная страница 'Недавние сессии' (Phase 3, epic forge-by9). Показывается в #main когда нет активных tmux-сессий (state.sessions.length===0 && !state.currentSession). Источник данных — backend Phase 2 (HistoryStore): GET /api/sessions/history (список), POST /api/sessions/history/restore {name,path}, POST /api/sessions/history/restore-all, DELETE /api/sessions/history {name,path}. Экспорты: renderHome() (рендер карточек/пустого состояния), restoreSession(name,path) (восстановление+открытие, 409-handling), restoreAll(), deleteHistory(name,path), showHome(show). DOM: контейнер #home в index.html (.home > .home-head[.home-title + #home-restore-all] + #home-cards + #home-empty), стили css/home.css (грид .home-cards, .home-card в палитре --bg-elev/--border-input, кнопки в стиле .btn-rename/.btn-kill). Интеграция: sessions.js::fetchSessions вызывает syncHomeVisibility (renderHome+showHome(true) при пустых сессиях, иначе showHome(false)); ws/attach.js onopen вызывает showHome(false) при открытии сессии. Цикл импортов home.js<->sessions.js безопасен (обращения только в рантайме внутри функций). XSS-safe рендеринг через document.createElement.
