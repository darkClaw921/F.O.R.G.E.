Фаза 2 (forge-joc): интеграция session_history в backend devforge (tmux-web/src/main.rs). 1) AppState.history: session_history::HistoryStore, init через HistoryStore::load(&themes_dir) (~/.config/forge/). 2) Периодический snapshot-воркер: tokio::spawn с tokio::time::interval(3600s), первый тик немедленно, на каждом тике session_history::capture_now(&history); в spawn передаётся только cheap-clone HistoryStore. 3) Shutdown-хук: клон HistoryStore захватывается в shutdown_signal future, после forge_echo::shutdown вызывается capture_now для финального снимка. 4) REST-роуты под auth: GET+DELETE /api/sessions/history, POST /api/sessions/history/restore, POST /api/sessions/history/restore-all. Общая логика восстановления — приватный хелпер restore_one_session.
