PWA Frontend (Фаза 4) — installability + офлайн app-shell для devforge. СТРОГИЙ OPT-IN: всё активно только при запуске сервера с флагом --pwa (backend GET /api/pwa/config отдаёт {enabled,vapidPublicKey} или 404).

Файлы:
- tmux-web/static/manifest.webmanifest — Web App Manifest: id '/', name 'F.O.R.G.E. — DevForge', short_name FORGE, start_url /?source=pwa, scope /, display standalone + display_override [standalone,minimal-ui], theme/background #0e1116, иконки 192/512 any + 192/512 maskable, shortcuts (Tasks /?tab=tasks, Э.Х.О /?tab=echo, Daily summary /?view=daily-summary), share_target GET → /?share-target=1 (title/text/url).
- tmux-web/static/sw.js — классический Service Worker, scope /. install: precache app-shell (/, style.css, vendor/xterm/*, /js/main.js + /quick-cmd.js + /command-dock.js + /hotkeys.js + /js/pwa/bootstrap.js + /icons/icon-192.png + /manifest.webmanifest) БЕЗ skipWaiting. activate: чистка forge-*-кэшей прошлых версий + clients.claim(). fetch (только GET): navigate → network-first+fallback кэш /; статика → stale-while-revalidate; read-only data allowlist (sessions/tasks/todos/echo conversations|memories|daily-reports) → network-first+cache fallback; /api/*+/healthz не кэшируются; /ws/* НИКОГДА не перехватывается. message SKIP_WAITING → skipWaiting. push → showNotification (icon-192/badge-72/vibrate/data.url). notificationclick → focus+navigate или openWindow. CACHE_VERSION='forge-pwa-v1'. data-партиционирование по хэшу токена — TODO (SW не имеет токена локально).
- tmux-web/static/js/pwa/bootstrap.js — ЕДИНСТВЕННЫЙ always-loaded PWA-модуль (подключён в index.html). fetch /api/pwa/config: не-200/enabled!=true → disablePwa() (unregister всех SW + delete forge-* кэшей, разметку не трогает); enabled → window.__FORGE_PWA={enabled,vapidPublicKey}, injectHead (manifest-link/theme-color/apple-meta/apple-touch-icon/pwa.css), import register.js → registerServiceWorker. Хук Фазы 5 (install/push/mobile.js + location.search) — TODO.
- tmux-web/static/js/pwa/register.js — registerServiceWorker(): register('/sw.js'), update-flow БЕЗ авто-skipWaiting: updatefound→installed+есть controller → баннер .pwa-update-banner «Обновить»; клик → reg.waiting.postMessage(SKIP_WAITING); controllerchange (guard refreshing) → один reload. Первая установка баннер не показывает.
- tmux-web/static/css/pwa.css — safe-area (--pwa-safe-* из env(safe-area-inset-*)), @media display-mode standalone/minimal-ui, .pwa-update-banner, .pwa-offline-banner (заготовка), .pwa-install-chip (заготовка). Инжектится только при enabled.
- tmux-web/static/index.html — viewport +viewport-fit=cover +interactive-widget=resizes-content; один <script type=module src=/js/pwa/bootstrap.js>. Статических manifest/apple/theme-color НЕТ (инжектит bootstrap).
- tmux-web/static/icons/ — icon-192/512.png (any), maskable-192/512.png, apple-touch-icon.png (180, без alpha), badge-72.png (моно), shortcut-tasks/echo/summary.png (96), favicon-32.png. Исходники: assets/icon.svg, assets/icon-maskable.svg, assets/badge.svg (rsvg-convert, ImageMagick нет — padding запечён в SVG).
- tmux-web/src/static_embed.rs — serve_static: для sw.js и manifest.webmanifest добавлен Cache-Control: no-cache (надёжный update-flow SW).
