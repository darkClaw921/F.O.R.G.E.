Фикс группировки tmux-сессий в sidebar по реальной leaf-папке cwd. До фикса — все сессии в подпапках стартового cwd форджа сливались под один project_id с именем стартовой папки. После — backend выдаёт folder_id/folder_label из basename(session.path), frontend (renderSidebar + renderOriginSection) группирует по folder_id. project_id оставлен для openSession→switchActiveProject и фильтров tasks/todos. План: ~/.claude/plans/resilient-seeking-cat.md. Затронуты файлы: tmux-web/src/main.rs (SessionDto, resolve_folder, get_sessions), tmux-web/static/app.js (renderSidebar, renderOriginSection, groupSessionsByFolder).
