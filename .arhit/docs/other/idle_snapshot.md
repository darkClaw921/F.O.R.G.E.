# idle_snapshot

AttentionState::idle_snapshot (tmux-web/src/attention.rs) — отдаёт «затихшие» сессии (HashMap<session, idle_secs>) для HostApi::idle_sessions (фича «Следующий шаг»). Источник — карта idle_started_at (метка ставится на фронте is_generating true→false в set_generating). ИСКЛЮЧЕНИЯ: (1) сессии с needs_attention=true (показан Claude prompt) — там нужен ответ, не следующий шаг; (2) сессии, которые пользователь смотрит непрерывно >= VIEW_DISMISS_SECS (10с, карта attached_since) — их idle-эпизод ЗАВЕРШАЕТСЯ: idle-метка удаляется насовсем, чтобы свечение погасло и не вернулось при уходе. Кроме того set_generating НЕ ставит idle-метку, пока сессия attached (перерисовка при переключении/просмотре != автономная генерация Claude) — это убирает баг «переключение сессий зажигает свечение». attached_since обновляется в watcher_loop каждый тик через update_attached(attached_names = сессии с attached>0).
