# git::is_valid_hash

Валидатор git-hash для защиты от инъекции аргументов git. Возвращает true ТОЛЬКО если длина строки в диапазоне 4..=64 И все байты — ASCII hex (0-9a-fA-F). Иначе false. Критичен, т.к. hash приходит из недоверенного query-параметра ?hash= хендлера get_git_commit и подставляется в командную строку git show — без валидации значение вроде '--output=...' или '; rm -rf /' могло бы быть истолковано git как флаг/команда. Диапазон длины покрывает сокращённые префиксы (от 4), полный SHA-1 (40) и SHA-256 (64). Вызывается первым делом в git::commit_detail; при false функция возвращает Ok(None) БЕЗ spawn git. Покрыт юнит-тестами (принимает реальные SHA и короткий префикс; отвергает пустую строку, '; rm -rf /', '../etc', не-hex, строку длиннее 64).
