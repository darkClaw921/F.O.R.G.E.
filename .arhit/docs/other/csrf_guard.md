# csrf_guard

Always-on anti-CSRF/drive-by middleware (tmux-web/src/auth.rs). Подключается в main.rs как внешний layer НЕЗАВИСИМО от remote/localhost режима (bearer_auth отсутствует в localhost). Защищает от drive-by RCE: 1) для /ws/* отвергает cross-origin WebSocket-апгрейд (Origin != Host) → 403, что закрывает атаку ws://127.0.0.1/ws/attach с вредоносного сайта; 2) для мутирующих /api/* (POST/PUT/PATCH/DELETE) требует same-origin Origin (если задан) и, при наличии тела, Content-Type: application/json (лишает атакующего 'простого запроса' без CORS-preflight). Хелперы: origin_authority (парсит host:port из Origin), is_same_origin (Origin==Host или Origin отсутствует), is_json_content_type, has_request_body (Content-Length>0 или Transfer-Encoding), is_mutating_method. Native-клиенты (CLI) без Origin пропускаются.
