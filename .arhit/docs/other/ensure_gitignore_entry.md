# ensure_gitignore_entry

Функция tmux-web/src/worktree.rs. Идемпотентно добавляет '.forge-worktrees/' в <toplevel>/.gitignore.

Назначение: скрывает каталог-контейнер вложенных рабочих копий от git, чтобы worktree внутри репозитория не засоряли 'git status' основного дерева. Модель размещения фичи — nested (<repo>/.forge-worktrees/<имя>/), поэтому без этой записи каждая worktree была бы видна как неотслеживаемый каталог.

Сигнатура: pub fn ensure_gitignore_entry(toplevel: &Path) (СИНХРОННАЯ, не async — чистый std::fs I/O).

Алгоритм:
1. Читает <toplevel>/.gitignore; если файла нет (NotFound) — считает содержимое пустым.
2. Если хотя бы одна строка (после trim, построчно) равна '.forge-worktrees/' — ничего не делает (идемпотентность).
3. Иначе дописывает '.forge-worktrees/\n', добавляя ведущий \n, если файл непустой и не заканчивается переводом строки (чтобы не склеить с предыдущей записью).

Параметры:
- toplevel: корень репозитория; целевой файл — <toplevel>/.gitignore.

Отказоустойчивость: НЕКРИТИЧНА для основного флоу. Любые ошибки чтения/записи (нет прав, гонка) не паникуют и не всплывают как Err, а лишь логируются через tracing::warn!. Отсутствие записи не ломает создание worktree — лишь делает каталог видимым в git status.

Связи: вызывается из create_worktree_window ДО worktree_add. Константы WORKTREES_DIR='.forge-worktrees' и GITIGNORE_ENTRY='.forge-worktrees/' определены в том же модуле. Покрыта unit-тестами (создание, идемпотентность, дозапись без хвостового newline).
