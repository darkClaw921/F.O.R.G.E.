# promote_todo

POST /api/todos/:id/promote (tmux-web/src/main.rs) — конвертирует TODO в bd-задачу и (если задана session) планирует notify-job. С Фазы 3 (forge-83eb) это тонкая HTTP-обёртка поверх promote_todo_core. Шаги: try_proxy_to_remote; парсинг PromoteTodoReq (session: Option, пустой body -> None); state.todos.get -> 404 если нет (ранний возврат остаётся в handler); target_session = req.session (trim/empty -> None; фолбэк на cfg.session делает core, НЕ handler); вызов promote_todo_core(state, &todo, target_session, mode_override=None) (режим вычисляет core из cfg); Err -> 400 BAD_REQUEST; ответ 200 { promoted: true, task_id, notify_scheduled }. Дефолтный текст уведомления при пустом cfg.template — DEFAULT_PROMOTE_TEMPLATE '[{id}] {title}' (только ID+заголовок, агент сам делает br show). plan-mode suffix дописывается отдельной строкой. Поведение HTTP не изменилось при рефакторинге в core.
