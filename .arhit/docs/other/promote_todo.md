# promote_todo

POST /api/todos/:id/promote — конвертирует TODO в bd-задачу (br create --json) и планирует notify-job в сессию. Дефолтный текст уведомления (когда NotifierConfig.template пуст) теперь '[{id}] {title}' (DEFAULT_PROMOTE_TEMPLATE) — в сессию уходит только ID и заголовок задачи, БЕЗ полного описания. Агент, получивший задачу, сам подтягивает описание через 'br show <id>'. Кастомные шаблоны с плейсхолдером {description} продолжают работать по-старому. plan-mode suffix дописывается отдельной строкой. Файлы: tmux-web/src/main.rs (promote_todo, format_notify_template, DEFAULT_PROMOTE_TEMPLATE), tmux-web/src/notifier_config.rs (NotifierConfig).
