# post_push_subscribe

Хендлер POST /api/push/subscribe (tmux-web/src/pwa.rs), Фаза 2 PWA. Тело = браузерный PushSubscription (subscription.toJSON()): { endpoint: String, keys: { p256dh, auth }, device_label?: Option<String> } — DTO SubscribeReq/SubscribeKeys (device_label через #[serde(default)], опционально). Сервер ставит created_at = Utc::now().to_rfc3339_opts(SecondsFormat::Secs, true) (RFC3339 UTC, секундная точность, заканчивается на Z; клиент НЕ задаёт таймстемп), собирает StoredSubscription и вызывает ctx.subs.upsert (дедуп по endpoint — повторный subscribe не плодит дубли, заменяет запись целиком включая ротированные keys). Ответ 200 { ok: true } (OkResp); ошибка записи на диск -> 500. На случай рассинхрона state.pwa == None -> 404. Регистрируется в main.rs только при app_state.pwa.is_some(), до auth-layer (в remote-mode под Bearer-auth + csrf_guard same-origin + application/json).
