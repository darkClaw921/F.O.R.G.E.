# main::try_proxy_to_remote

Phase 3 helper в main.rs — общая ветка ?server=<id> для resource handler'ов. Сигнатура: async fn try_proxy_to_remote(state, raw_query, method, path, content_type, body, enrich_array) -> Option<Result<Response, (StatusCode, String)>>. Логика: (1) если в query нет 'server' или пустой — возвращает None (handler продолжает локальную логику); (2) если ?server=<id> есть, но state.remote_mode == false — возвращает Some(Err((400, 'remote mode disabled — cannot use ?server=<id>'))); (3) иначе вызывает remote_proxy::proxy_request на target. Параметр enrich_array=true означает, что JSON-тело должно быть обогащено origin=<server_id> через remote_proxy::enrich_with_origin (только для Content-Type: application/json). Используется во ВСЕХ ресурсных handler'ах: get_sessions, create_session, delete_session, get_projects, get_tasks (специальная обработка для {issues:[...]}), create_task, patch_task, close_task, reopen_task, get_todos, create_todo, patch_todo, delete_todo, promote_todo. Сопутствующие хелперы: extract_server_id (trim + filter empty), rebuild_query_without_server (drops 'server' key), urlencode_minimal (для path-сегментов id), proxy_response_to_axum (reqwest::Response -> axum::Response, фильтрация hop-by-hop, опциональное обогащение JSON через enrich_with_origin).
