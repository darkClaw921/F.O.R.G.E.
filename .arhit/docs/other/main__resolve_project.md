# main::resolve_project

Возвращает только project_id и project_name для tasks/scope-linking. НЕ отвечает за sidebar-группировку — для этого есть отдельный helper resolve_folder. Логика: longest-prefix-match по зарегистрированным проектам в Snapshot (по polевой нормализации path) → tmux_prefix (если session.tmux_prefix совпадает с project.tmux_prefix) → fallback на синтетический ключ '__path__:<cwd>'. Если path пуст — (None, None) (orphan). Используется внутри замыкания .map() в get_sessions (main.rs ~807-822). Результат идёт в SessionDto.project_id / SessionDto.project_name, которые фронт затем потребляет в switchActiveProject (через openSession) и в фильтрах TODO/.beads. После Phase 2/3 группировка sidebar НЕ зависит от project_id — она построена на folder_id (см. resolve_folder).
