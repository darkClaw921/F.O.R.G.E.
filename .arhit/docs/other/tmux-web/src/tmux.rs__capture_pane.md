# tmux-web/src/tmux.rs::capture_pane

Захват видимой части активной панели tmux-сессии (tmux capture-pane -p -J -t <session>:). КРИТИЧНО: target формируется как format!("{session}:") с суффиксом-двоеточием — без него tmux для ЧИСЛОВЫХ имён сессий (0,1,2,…) резолвит -t 1 как ОКНО №1 текущей активной сессии, а не как сессию с именем '1'. Этот баг приводил к дублированию индикатора работы (✶) и оранжевого свечения needs_attention: watcher сессии '1' захватывал чужой pane (окно сессии '0', у которой 2 окна) и детектил то же содержимое. Симметрично исправлено в capture_pane_full. Используется attention::watcher_loop.
