# tmux-web/static/app.js::renderOriginSection

Phase 6 — рендерит ОДНУ origin-секцию в session-list: header (свёрнут/развёрнут с caret и dot-индикатором online/offline) → projects (sub-header) → sessions (item с in-origin классом). Параметры: originKey ('local' | server_id), label (текст header'а), dotKind ('online'|'offline'|'local'|'unknown'), projects (массив ProjectDto), sessions (массив SessionDto), opts.isRemote, opts.isOffline (если true — секция в disabled-стиле, без списка сессий, с бейджем 'offline' вместо счётчика), opts.remoteLoading (true → плейсхолдер 'Loading…'). Внутри origin'а применяет такую же группировку sessions по project_id как legacy renderSidebar при projectFilter=='__all__': projects в порядке передачи → авто-группы по project_id (которого нет в projects) → orphan (project_id=null). Двухуровневая фильтрация: если state.projectFilter !== '__all__' — показывает только сессии этого проекта внутри origin'а, без sub-header'а (как legacy single-project режим). Если проекта в origin'е нет — пустая секция с пометкой.
