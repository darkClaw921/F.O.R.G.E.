# app.js::renderOriginSection

Рендерит одну origin-секцию в session-list (header origin'а + список сессий, сгруппированных по folder_id). Сигнатура: renderOriginSection(originKey, label, dotKind, projects, sessions, opts). Параметр projects сохранён для backward-совместимости, но больше не используется для построения header'ов (folder_id формата __folder:<path> не матчится с p.id). Группировка: visible = projectFilter !== '__all__' ? sessions.filter(s.project_id===pf) : sessions; затем groupSessionsByFolder(visible, ORPHAN_KEY). Header группы = arr[0].folder_label || basename(key.slice('__folder:'.length)). Сортировка ключей по folder_label (case-insensitive localeCompare), Orphan-группа в конце. opts.isOffline — рендерит badge 'offline' и плейсхолдер 'Сервер недоступен'. opts.remoteLoading — 'Loading…'. Пустой visible → empty placeholder ('Нет сессий в этом проекте' если projectFilter активен, иначе 'Нет активных сессий'). openSession→switchActiveProject продолжает работать через sess.project_id. Файл: tmux-web/static/app.js (~785–935). Phase 3: переписан с registered-projects header'ов на единый folder-based проход (синхронно с legacy renderSidebar).
