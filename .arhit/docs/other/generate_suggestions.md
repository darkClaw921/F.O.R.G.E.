# generate_suggestions

Генерирует предложения задач по проектам через отдельный one_shot. Вход: весь контекст дня (сообщения чатов, snapshot tmux-панелей, per-project git-активность). Если проектов нет — runner не вызывается, возвращается пустой вектор. Эффективный промпт предложений читается из app_settings по ключу PROMPT_KEY_SUGGEST (daily_report.suggest_prompt): пользовательский оверрайд используется только если непустой (filter по trim), иначе фолбэк на дефолтную константу SUGGEST_META_PROMPT. Любая ошибка генерации/парсинга деградирует до пустого вектора (основной отчёт важнее). Константы REPORT_META_PROMPT/SUGGEST_META_PROMPT и ключи PROMPT_KEY_REPORT/PROMPT_KEY_SUGGEST имеют видимость pub(crate) — нужны REST-слою (Phase 3) для отдачи дефолтов и чтения/записи оверрайдов.
