# next_step

Воркер фичи «Следующий шаг» (plugins/echo/src/next_step/mod.rs). Каждые TICK_INTERVAL (2с) опрашивает HostApi::idle_sessions(); для затихших сессий с idle_secs >= IDLE_THRESHOLD_SECS (10с) генерирует РОВНО ОДИН короткий следующий шаг через state.runner.one_shot. Prompt = NEXT_STEP_META_PROMPT (RU, требует один шаг без markdown/преамбул) + подмешанные правила памяти next_step_rules::list_rules + последние 100 строк pane (capture_pane_full). Результат кладётся в state.next_steps[session] (NextStepSuggestion) и рассылается broadcast'ом ServerMsg::NextStepEvent{session, has_suggestion:true}. Защита от двойного запуска: in-memory ProcessedSet (Arc<Mutex<HashSet<String>>>) + проверка наличия в state.next_steps. Сброс эпизода: при исчезновении сессии из idle-списка (снова активна/prompt/закрыта) reset_stale_episodes чистит processed + next_steps и шлёт NextStepEvent{has_suggestion:false}. Пустой ответ модели не сохраняется. spawn(state, host)->JoinHandle регистрируется в crate::spawn_workers для graceful shutdown. Ключевые функции: spawn, run_loop, tick_once (pub(crate), тестируемый), reset_stale_episodes, generate_for_session.
