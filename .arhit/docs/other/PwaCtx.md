# PwaCtx

PWA-контекст (tmux-web/src/main.rs), opt-in под флагом --pwa. Живёт в AppState.pwa как Option<PwaCtx>: Some => PWA включён (роуты /api/pwa/* + /api/push/* зарегистрированы, файлы ~/.forge/{vapid.json,push_subscriptions.json} созданы/загружены); None => поведение байт-в-байт как раньше (жёсткий opt-in инвариант, 404 на все PWA-роуты, никаких файловых операций в ~/.forge). Cheap-clonable. Поля: vapid: vapid::VapidStore (VAPID P-256 пара — публичный отдаётся через GET /api/pwa/config, приватный для ES256-JWT в Фазе 3); subs: push_store::PushSubscriptionStore (Фаза 2 — хранилище push-подписок ~/.forge/push_subscriptions.json, пишется эндпоинтами subscribe/unsubscribe, читается push-воркером доставки Фазы 3). Инициализируется в main.rs внутри блока if pwa_enabled: резолв default_vapid_path() + create_dir_all(~/.forge) + VapidStore::load_or_generate, затем default_subscriptions_path() + PushSubscriptionStore::new. Любая ошибка резолва/загрузки -> warn + старт без PWA (None), не fail-fast (терминал/WS важнее опционального PWA).
