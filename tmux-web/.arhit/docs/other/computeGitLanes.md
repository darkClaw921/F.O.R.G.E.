# computeGitLanes

Чистая функция в static/app.js, вычисляющая lane-layout для коммит-графа. Сигнатура: computeGitLanes(commits) -> { commits, edges, maxLanes }. Алгоритм: проход сверху вниз (newest first); lanes[] — массив открытых hashes по столбцам (null = свободный слот). Для каждого commit: 1) ищет свою колонку через lanes.indexOf(commit.hash) или берёт null-слот / push новой; 2) аннотирует commit.lane и commit.row; 3) накапливает edges для каждого parent; 4) первый parent наследует колонку коммита, остальные занимают свободные слоты или push (octopus merge); 5) если parent уже в lanes — это merge-down, новая колонка не выделяется. Второй проход резолвит edges: toRow и toLane по hashIndex; parent вне видимости даёт обрыв до низа canvas. Edges формат: { fromLane, fromRow, toLane, toRow }. maxLanes >= 1 даже на пустом списке. Переиспользование null-слотов уменьшает разрастание maxLanes.
