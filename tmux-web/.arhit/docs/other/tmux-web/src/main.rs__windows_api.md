# tmux-web/src/main.rs::windows_api

REST-эндпоинты управления окнами tmux внутри сессии. Все поддерживают origin-проксирование через try_proxy_to_remote(?server=<id>).\n\n- GET /api/sessions/:name/windows → JSON [WindowInfo]. Хендлер list_windows. 400 если сессии нет.\n- POST /api/sessions/:name/windows → 201 Created. Body опционален: { name?: string }. Пустое тело → дефолтное имя tmux. Хендлер create_window.\n- POST /api/sessions/:name/windows/:index/select → 204 No Content. Делает указанное окно активным; прикреплённые WS-клиенты следуют автоматически. Хендлер select_window.\n- DELETE /api/sessions/:name/windows/:index → 204 No Content. Хендлер delete_window.\n- PATCH /api/sessions/:name/windows/:index → 200 + { name } — переименовывает окно. Body: { name: string }. Хендлер patch_window.\n\nAxumPath<(String, u32)> используется для парсинга составного пути с index. Все ошибки → 400 Bad Request с stderr-сообщением tmux.
