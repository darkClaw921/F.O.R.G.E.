# tmux-web/src/tmux.rs::windows

Окно tmux (window) внутри сессии — функции listing/CRUD.\n\n- pub struct WindowInfo { index: u32, name: String, active: bool, panes: u32 } — DTO одного окна. active=true для текущего активного окна сессии (#{window_active}=1). panes — кол-во панелей внутри окна.\n- const LW_FORMAT — формат для tmux list-windows: '#{window_index}|#{window_name}|#{window_active}|#{window_panes}'.\n- pub async fn list_windows(session) → Vec<WindowInfo>. В отличие от list_sessions ошибка 'no server running' / 'can't find session' возвращается как Err (вызывающий специально запрашивал окна конкретной сессии).\n- fn parse_window_line(line) → Option<WindowInfo> — парсит одну строку, splitn(4, '|'). active=matches!(_, '1').\n- pub async fn new_window(session, name: Option<&str>) — 'tmux new-window -t <session>' (+ '-n <name>' если задано). tmux назначает следующий свободный index и сразу делает окно активным.\n- pub async fn select_window(session, index) — 'tmux select-window -t <session>:<index>'. Все прикреплённые WS-клиенты автоматически переключаются.\n- pub async fn kill_window(session, index) — 'tmux kill-window -t <session>:<index>'. Если это последнее окно — сессия тоже умирает.\n- pub async fn rename_window(session, index, name) — 'tmux rename-window -t <session>:<index> <name>'. Пустое имя → Err.\n\nВсе функции валидируют session через is_valid_session_name. index типизирован как u32 (axum-extractor парсит автоматически).
