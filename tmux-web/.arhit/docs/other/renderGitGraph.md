# renderGitGraph

Функция в static/app.js, рисующая коммит-граф на canvas #git-graph-canvas. Вызывается из fetchGitLog после обновления state.gitLog. Алгоритм: 1) пустой лог — placeholder 'No commits' (240x40 px); 2) computeGitLanes(state.gitLog) даёт layout; 3) расчёт logical размера canvas: width = maxLanes*LANE_W + META_W (600), height = N*ROW_H + 16; 4) HiDPI scaling: атрибуты width/height = logical*dpr, style.width/height = logical px, ctx.setTransform(dpr,0,0,dpr,0,0); 5) рисует edges первыми (под узлами): прямая вертикаль если fromLane===toLane, иначе bezier через две контрольные точки на mid-Y; цвет = palette[toLane%len]; 6) узлы — filled circle радиус 5px, цвет = palette[lane%len]; HEAD-узел (refs содержит 'HEAD' / 'HEAD -> *') получает белую обводку толщиной 2px; 7) метаданные справа: abbrev + (refs) + subject, ctx.font='12px monospace', цвет #d8dee9; subject обрезается с многоточием через бинарный поиск по measureText. Константы: GIT_LANE_W=18, GIT_ROW_H=24, GIT_NODE_R=5, GIT_META_W=600, GIT_LANE_PALETTE — массив 7 цветов.
