# purge_task

Backend handler tmux-web/src/main.rs. POST /api/tasks/:id/purge — физически удаляет issue через 'br delete --hard --force --json --reason clean-column <id>'. Маршрут зарегистрирован в .route("/api/tasks/:id/purge", post(purge_task)). Поддерживает remote-proxy через try_proxy_to_remote. cwd берётся из активного проекта. Успех → 204 No Content. ИДЕМПОТЕНТНОСТЬ (фикс UX bulk-clean): если br возвращает ISSUE_NOT_FOUND (issue уже удалён или никогда не существовал) — handler возвращает 204 вместо 400. Это устраняет ложные ошибки при гонке bulk-clean с WS-обновлениями state. Использован bulk-clean кнопкой frontend (forge-caz3) для очистки колонки Closed.
